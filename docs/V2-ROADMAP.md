# 宝宝喝奶记录 V2 - 功能规划

## 当前版本（V1）能力总结

V1 实现了核心的多人协作喂奶记录功能：快捷记录、历史查看、家庭码共享、基础统计。数据存储在 Upstash Redis，部署在 Vercel，支持 PWA。

---

## V2 目标

在 V1 的基础上，从「能用」升级到「好用」。重点方向：

1. **数据洞察** — 让家长真正理解宝宝的喂养规律
2. **多维记录** — 不只是喝奶，覆盖更多育儿场景
3. **体验优化** — 更流畅的交互和更可靠的离线能力
4. **安全与协作** — 保护数据安全，提升多人协作体验

---

## 功能模块

### 一、数据统计与可视化

**问题：** V1 仅有今日次数和总量两个数字，家长无法发现喂养趋势和规律。

**V2 方案：**

- 每日/每周/每月奶量趋势折线图
- 喂奶间隔分布图（帮助发现宝宝的喂养节奏）
- 单次奶量变化趋势（观察宝宝食量增长）
- 各记录人贡献占比（谁喂得多一目了然）
- 支持按日期范围筛选
- 数据导出为 CSV/图片，方便分享给医生

### 二、多类型记录

**问题：** V1 只能记录喝奶，但新生儿护理涉及更多维度。

**V2 方案：**

- 新增记录类型：
  - 🍼 母乳亲喂（记录时长而非奶量）
  - 💧 喝水
  - 🍌 辅食（适用于 6 月龄以上）
  - 💩 换尿布（大便/小便/混合）
  - 😴 睡眠（入睡/醒来）
  - 📏 身高体重（定期测量）
- 首页根据记录类型显示不同的统计卡片
- 历史页支持按类型筛选

### 三、提醒与通知

**问题：** V1 只能被动查看距离上次喂奶的时间，容易错过喂奶时间。

**V2 方案：**

- 可设置喂奶间隔提醒（如每 3 小时提醒一次）
- 浏览器 Push Notification 推送
- 提醒时显示距离上次喂奶的时间和奶量
- 家庭成员可各自设置是否接收提醒
- 提醒静默时段设置（如夜间免打扰）

### 四、离线优先与数据同步

**问题：** V1 的乐观更新在网络不稳定时可能丢失数据，且无 Service Worker。

**V2 方案：**

- 引入 Service Worker，实现真正的离线可用
- 离线操作队列：断网时记录操作，恢复后自动同步
- 冲突检测与解决策略（基于时间戳的 last-write-wins）
- 同步状态指示器（已同步/同步中/待同步）
- 静态资源缓存，提升加载速度

### 五、安全与权限

**问题：** V1 的家庭码是唯一的访问凭证，无鉴权机制，任何知道 familyId 的人都能读写数据。

**V2 方案：**

- 家庭码加入时需要创建者确认（邀请审批）
- 为每个家庭成员生成 token，API 请求携带 token 鉴权
- 记录操作日志（谁在什么时候做了什么）
- 支持移除家庭成员
- 家庭码可重新生成（旧码失效）
- 数据定期自动备份

### 六、协作体验增强

**问题：** V1 的多人协作仅限于共享数据，缺乏实时感知。

**V2 方案：**

- 新记录实时推送到其他家庭成员（WebSocket 或轮询）
- 记录附带操作人头像/颜色标识
- 家庭成员列表页面（查看谁加入了家庭）
- 记录备注功能（如「吐了一点」「喝得很快」）

### 七、UI/UX 优化

**V2 方案：**

- 深色模式支持
- 首页下拉刷新手势
- 记录列表虚拟滚动（大量数据时保持流畅）
- 添加记录成功的动画反馈
- 引导式新手教程
- 多语言支持（中/英）
- 宝宝头像上传

### 八、成长时间线

**V2 方案：**

- 按周/月展示宝宝成长里程碑
- 自动生成周报/月报摘要（本周喝了多少奶、平均间隔等）
- 支持添加自定义里程碑（第一次翻身、第一颗牙等）
- 可分享的成长卡片（生成图片）

## 优先级排序

| 优先级 | 功能模块 | 理由 |
|--------|----------|------|
| P0 | 离线优先与数据同步 | 基础体验保障，数据不能丢 |
| P0 | 安全与权限 | 涉及婴儿数据，安全不能妥协 |
| P1 | 数据统计与可视化 | 用户最直接的价值感知 |
| P1 | 提醒与通知 | 高频需求，直接影响使用粘性 |
| P2 | 多类型记录 | 扩展产品边界，但需要较大改动 |
| P2 | 协作体验增强 | 提升多人场景体验 |
| P3 | UI/UX 优化 | 锦上添花，可逐步迭代 |
| P3 | 成长时间线 | 情感价值高，但非核心功能 |

## 技术升级建议

| 方向 | 建议 | 理由 |
|------|------|------|
| 状态管理 | 引入 Zustand 或 Jotai | 当前 useState + prop drilling 在功能增多后难以维护 |
| 数据获取 | 引入 TanStack Query | 统一管理缓存、重试、乐观更新、后台刷新 |
| 图表库 | Recharts 或 Chart.js | 轻量级，满足统计可视化需求 |
| 离线 | Workbox | Google 官方 Service Worker 工具库 |
| 实时通信 | Upstash Redis Pub/Sub 或 SSE | 复用现有 Upstash 基础设施 |
| 测试 | Vitest + Testing Library | 核心逻辑需要测试覆盖 |
| 数据库 | 考虑迁移到 Upstash Redis + 关系型数据库 | 记录量增长后 KV 存储的查询能力有限 |
