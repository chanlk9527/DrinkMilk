# 宝宝 Care App — 全方位功能规划

## 背景

当前产品以喝奶记录为核心功能，已实现疫苗记录和成长曲线追踪。为了从「喂奶工具」升级为「全方位宝宝 Care App」，本文档梳理了可扩展的功能方向。

---

## 已实现功能

- ✅ 喝奶记录（奶量、时间、记录人）
- ✅ 成长记录（身高、体重、头围 + WHO 生长曲线）
- ✅ 疫苗记录
- ✅ 数据统计与可视化
- ✅ 多人协作（家庭码共享）
- ✅ 深夜模式

---

## 功能规划

### 一、日常记录类

高频操作，是宝宝 Care App 的基础能力。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 🩲 换尿布记录 | 记录次数、类型（大便/小便/混合）、颜色、性状，帮助判断消化健康 | P0 |
| 😴 睡眠记录 | 入睡/醒来时间、睡眠时长统计、日夜睡眠比例分析 | P0 |
| 🥣 辅食记录 | 食材种类、进食量、过敏反应追踪（6 月龄+） | P1 |
| 💧 喝水记录 | 饮水量追踪（辅食期后） | P2 |
| 🌡️ 体温记录 | 日常体温监测，异常体温高亮提醒 | P1 |

### 二、成长发育类

帮助家长了解宝宝发育进度，情感价值高。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 🏅 发育里程碑 | 追踪翻身、坐、爬、站、走、说话等关键里程碑，对比标准月龄范围 | P1 |
| 📸 成长相册 | 按月龄自动归档照片，支持添加文字记录，生成成长时间线 | P2 |
| 📝 成长日记 | 每日/每周自由文字记录，记录宝宝的趣事和变化 | P2 |
| 🦷 出牙记录 | 记录每颗牙的萌出时间，可视化牙齿图 | P3 |

### 三、健康管理类

保障宝宝健康，提供就医参考。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 💊 用药记录 | 药品名称、剂量、频次、用药周期追踪 | P1 |
| 🏥 就诊记录 | 医院、医生、诊断结果、处方记录，方便复诊参考 | P2 |
| ⚠️ 过敏原记录 | 记录已确认的过敏原（食物、药物、环境），全局提醒 | P1 |
| 📋 体检记录 | 定期体检数据归档，发育评估结果记录 | P2 |

### 四、提醒类

从被动记录到主动提醒，提升使用粘性。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| ⏰ 喂奶提醒 | 基于上次喂奶时间，自定义间隔提醒 | P1 |
| 🩲 换尿布提醒 | 超过设定时间未换尿布时提醒 | P2 |
| 💉 疫苗接种提醒 | 根据宝宝月龄自动推算下次疫苗时间 | P1 |
| 🩺 体检提醒 | 按国家标准体检时间表提醒 | P2 |
| 🍼 辅食引入提醒 | 到达辅食添加月龄时提醒，附带注意事项 | P3 |

### 五、数据分析类

让数据产生洞察，帮助家长科学育儿。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 📊 每日/每周摘要 | 自动生成喂养、睡眠、排便等综合报告 | P1 |
| 🔗 喂养-睡眠关联 | 分析喂奶量/时间与睡眠质量的关系 | P2 |
| 📈 排便规律分析 | 统计排便频率、时间分布，发现异常 | P2 |
| 👨‍👩‍👧 多看护人统计 | 各家庭成员的照护贡献统计 | P3 |
| 📤 数据导出 | 导出为 PDF/CSV，方便分享给医生 | P2 |

### 六、实用工具类

提升日常育儿便利性。

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 🎵 白噪音/安抚音乐 | 内置多种安抚音效（雨声、心跳、吹风机等） | P3 |
| 📋 出行清单 | 预设外出物品清单模板，可自定义勾选 | P3 |
| 🧮 奶量计算器 | 根据月龄和体重推荐每日奶量参考值 | P2 |
| 📖 育儿知识库 | 按月龄推送喂养、护理、发育相关知识 | P3 |

## 建议实施路线

### 第一阶段：核心记录扩展

重点补齐高频日常记录能力，与现有喂奶记录形成完整的日常护理闭环。

1. 换尿布记录 — 实现逻辑与喂奶记录类似，开发成本低
2. 睡眠记录 — 新手爸妈最关心的话题，支持计时器模式
3. 体温记录 — 简单但实用，健康管理的基础数据

### 第二阶段：健康与发育

在日常记录的基础上，增加健康管理和发育追踪能力。

1. 过敏原记录 — 辅食期的刚需
2. 辅食记录 — 与过敏原联动
3. 发育里程碑 — 情感价值高，用户粘性强
4. 用药记录 — 生病时的刚需

### 第三阶段：智能化与体验

让 App 从「记录工具」升级为「智能助手」。

1. 提醒系统（喂奶、疫苗、体检）
2. 每日/每周摘要报告
3. 数据导出
4. 喂养-睡眠关联分析

### 第四阶段：情感与社区

提升产品的情感价值和长期留存。

1. 成长相册与时间线
2. 成长日记
3. 育儿知识库
4. 白噪音/安抚音乐

## 数据模型扩展参考

```typescript
// 换尿布记录
interface DiaperRecord {
  id: string
  at: string              // ISO 8601
  type: 'pee' | 'poop' | 'mixed'
  color?: string          // 大便颜色
  consistency?: string    // 性状（稀/软/硬）
  by: string
  notes?: string
}

// 睡眠记录
interface SleepRecord {
  id: string
  startAt: string         // 入睡时间
  endAt?: string          // 醒来时间（进行中则为空）
  quality?: 'good' | 'fair' | 'poor'
  by: string
  notes?: string
}

// 辅食记录
interface SolidFoodRecord {
  id: string
  at: string
  food: string            // 食材名称
  amount?: string         // 进食量描述
  reaction?: 'none' | 'mild' | 'severe'  // 过敏反应
  by: string
  notes?: string
}

// 体温记录
interface TemperatureRecord {
  id: string
  at: string
  tempC: number           // 摄氏度
  method: 'ear' | 'forehead' | 'armpit' | 'rectal'
  by: string
  notes?: string
}

// 用药记录
interface MedicationRecord {
  id: string
  at: string
  name: string            // 药品名称
  dosage: string          // 剂量
  by: string
  notes?: string
}

// 发育里程碑
interface MilestoneRecord {
  id: string
  date: string            // YYYY-MM-DD
  type: string            // 里程碑类型（翻身/坐/爬/走/说话等）
  description?: string
  photoUrl?: string
}
```

## 技术考量

- 新增记录类型复用现有的存储和同步架构（Upstash Redis + Google Drive）
- 每种记录类型独立存储 key，避免单 key 数据量过大
- 提醒功能依赖浏览器 Notification API + Service Worker
- 相册功能需要考虑图片存储方案（压缩 + 云存储）
- 数据模型设计保持向后兼容，新增字段均为可选
